# Content Structuring

## When To Use

- Writing or updating build descriptions and progression notes.
- Standardizing labels, summaries, and comparison text across games.
- Preparing seeded content for cards, detail pages, advisor explanations, and dashboards.

## Steps

1. Write a one-line fantasy for the build identity.
2. Add concise summary, playstyle, and mode descriptors.
3. Break strengths, weaknesses, and progression into short structured entries.
4. Tag each build with mechanics, pace, difficulty, and audience signals.
5. Ensure content can support both compact cards and long-form detail pages.

## Standards

- Content should be informative first and stylish second.
- Avoid filler lore if it does not help a player choose or understand the build.
- Each build needs enough differentiation to compare against alternatives.
- Keep wording crisp so it fits cards, advisor output, and dashboard labels.
